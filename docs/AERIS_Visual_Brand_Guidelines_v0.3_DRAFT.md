**SECTION 01**

**About this guide**

This is the source of truth for how AERIS looks.

It covers the logo, colour, typography, eyebrow labels, layout, iconography, imagery, and buttons. Use it for marketing, product UI, support content, sales material, partner content, and anything that carries the AERIS name.

This guide is for designers, product writers, marketers, support, partners, and any external contributor to AERIS. Keep it close, apply it consistently and when in doubt, the gut check at the end will sort most things out.

**SECTION 02**

**The visual identity in one line**

| AERIS LOOKS LIKE Confident. Practical. Quiet. Built for work. *A visual system that earns trust by being clear and consistent, the same way every time.* |
| :---- |

Four traits hold the visual identity together. None of them works on its own. A layout that is quiet but flat is dull. A layout that is confident but loud is pushy. The identity lives where all four meet.

**01  CONFIDENT**

The system knows what it is. No effects, no flourishes, no apologetic flourish to soften it. It says what it is, the same way every time.

**02  PRACTICAL**

Every element does work. Colour signals hierarchy, type signals importance, icons signal action, and nothing sits on the page for decoration.

**03  QUIET**

The brand doesn’t shout. It uses whitespace, calm pairings, and one typeface. Restraint is the move.

**04  CONSISTENT**

The system holds up across SMB marketing, enterprise documents, partner co-branding, and the product itself. Same logo, same colours and same rules.

**SECTION 03**

**The logo**

AERIS uses a custom wordmark. It’s the primary identifier of the brand and carries the most visibility of any element in the system.

The wordmark is a custom logotype. It’s not Poppins typed out, and it can’t be recreated by typing AERIS in any font. Always use the production files.

| PRODUCTION FILES [AERIS BRANDING folder, Google Drive](https://drive.google.com/drive/folders/1Ru5JiuS8aA-KiDebmxkyYipV3z3aFjY_?usp=sharing) |
| :---- |

## **Variants**

Four core variants. Use the one that gives the highest contrast against its background.

* Light background, solid

* Light background, transparent

* Dark background, solid

* Dark background, transparent

## **Clear space**

Leave clear space around the wordmark equal to the height of the A. Nothing else, including type, photography, or graphic devices, sits inside this zone.

## **Minimum size**

* Digital: 24 px height

* Print: 15 mm height

Below these sizes, the wordmark loses legibility. Omit the mark or use the monogram instead.

## **Placement**

Top-left for digital. A consistent corner anchor across a campaign for print. Centred placement is reserved for symmetrical layouts. The wordmark sits on flat colour or quiet areas of an image, not on competing detail.

## **Logo on photography**

On busy photography, set the wordmark on a flat colour panel, a soft scrim, a cropped colour block from the palette, or choose a monochrome logo. Test contrast at the smallest expected size.

**SECTION 04**

**Colour**

The AERIS palette is the core of the visual system and remains locked under the new identity. There are five colours. Two primaries, two accents and one neutral. Use them with intent.

|  | Royal Red Role: PRIMARY HEX \#780000    RGB 120 / 0 / 0 *Strong brand statements. Hero panels, key moments, dominant headings.* |
| :---- | :---- |
|  | **Red Dirt Red** **Role: ACCENT** HEX \#C1121F    RGB 193 / 18 / 31 *Calls to action, active states, eyebrow labels. Used sparingly to draw the eye.* |
|  | **Loyal Navy** **Role: PRIMARY** HEX \#003049    RGB 0 / 48 / 73 *Default text, headings, structural elements. The workhorse of the system.* |
|  | **Dusty Blue** **Role: SECONDARY ACCENT** HEX \#669BBC    RGB 102 / 155 / 188 *Supporting elements, secondary buttons, soft highlights, charts and data.* |
|  | **Clermont Cream** **Role: NEUTRAL** HEX \#FDF0D5    RGB 253 / 240 / 213 *Default background. Pairs with Loyal Navy for the highest-readability setting.* |

## **Usage hierarchy**

* **Primary:** Royal Red and Loyal Navy carry brand recognition. They dominate.

* **Accent:** Red Dirt Red and Dusty Blue highlight and signal action. Used sparingly.

* **Neutral:** Clermont Cream provides balance. It’s the default background for legibility.

## **Default pairings**

* Loyal Navy text on Clermont Cream. The high-readability default for long-form reading.

* Royal Red on Clermont Cream. Headings and accent text.

* Clermont Cream on Loyal Navy. Inverted setting for hero panels and dark sections.

* Red Dirt Red on Clermont Cream. Eyebrow labels, calls to action, active states.

* White on Red Dirt Red. Primary buttons.

## **Accessibility**

All text combinations meet WCAG 2.1 AA contrast: 4.5:1 for body text, 3:1 for large text. Don’t pair Royal Red with Red Dirt Red, or Dusty Blue with Clermont Cream, as foreground and background.

## **Print and digital**

Use the hex values for digital. For print, convert to CMYK or Pantone via a calibrated proof. Stick to brand-approved hex codes for uniformity. Don’t eyeball new shades.

## **Hierarchy**

Use the scale below as the starting point. Adjust for the medium, but keep the relative steps between levels intact.

| Style | Weight | Size | Use |
| :---- | :---- | :---- | :---- |
| **Display / H1** | Bold (700) | 32-48 pt | Hero headings, campaign moments |
| **H2** | SemiBold (600) | 24-28 pt | Section titles |
| **H3** | Medium (500) | 18-20 pt | Sub-sections, card titles |
| **Body** | Regular (400) | 11-12 pt | Default reading text |
| **Body emphasis** | Medium (500) | 11-12 pt | In-text emphasis, lead-ins |
| **Caption** | Regular (400) | 9-10 pt | Labels, metadata, fine print |
| **Button / UI** | SemiBold (600) | 12-14 pt | Buttons, navigation, UI labels |

## **Case**

Sentence case for headings. Not Title Case That Looks Like A Brochure.

* Sentence case for all headings and sub-headings.

* UPPER CASE is reserved for eyebrow labels and short tags.

* Title Case is acceptable for product names, proper nouns, and the AERIS wordmark only.

## **Alignment and rhythm**

* Left-aligned by default. Centred only for hero moments and short statements.

* Line height: 1.4 to 1.6 for body. 1.1 to 1.3 for headings.

* Avoid orphans at the end of headings and breakouts.

## **Web fallback**

"Poppins", "Helvetica Neue", Arial, sans-serif

**SECTION 06**

**Eyebrow labels**

Eyebrow labels are short, all-caps tags in Red Dirt Red. They introduce a section, set context, or signal a category. Used well, an eyebrow does the work of a heading and an explanation in three words.

| BUILT IN CLERMONT QLD, DEPLOYED EVERYWHERE Run your entire business from one system |
| :---- |

## **Format**

* Poppins SemiBold, all caps, letterspaced 4 to 6 percent.

* Red Dirt Red on light backgrounds. Clermont Cream on dark.

* Sized small: 10 to 12 pt print, 12 to 14 px digital.

* One eyebrow per section. Never decorative.

## **Use them for**

* Section openers above a heading.

* Labels under hero images.

* Product status tags. MOST POPULAR. NEW. BETA.

* Editorial breakouts. THE HIDDEN COST.

## **Don’t use them for**

* Body copy emphasis. Use bold instead.

* Decoratively breaking up text.

* Long phrases over six words.

**SECTION 07**

**Layout**

AERIS layouts are built on three things: cards, panels, and whitespace.

## **Cards**

Information is grouped into rounded card containers. Cards have:

* 24 to 32 px corner radius.

* Generous internal padding. 32 to 48 px on desktop.

* One purpose per card.

Cards separate distinct ideas. They aren’t decoration.

## **Panels**

A cream panel signals a feature moment. A navy panel signals contrast or emphasis. A red panel signals a call to action or a critical point. Panels can be full-bleed or contained.

| CREAM PANEL Feature moments, hero treatments, the default light surface. | NAVY PANEL Emphasis, callouts, inverted hero panels. | RED PANEL Calls to action, critical points, urgency. |
| :---- | :---- | :---- |

## **Whitespace**

AERIS layouts breathe. Headings have room above them. Cards don’t crowd each other. The reader gets to the point without fighting the page. If a layout feels crowded, remove something. Don’t add.

## 

## **Grid**

* 12-column grid on desktop.

* 4-column on mobile.

* Standard column gutters.

**SECTION 08**

**Iconography**

Icons should support the interface, not dominate it. AERIS icons are line-based and geometric. They earn their place by being useful.

## **Style**

* Line icons, 1.5 to 2 px stroke at 24 px default size.

* Rounded line caps and joins.

* Built on a 24 px artboard with a 2 px safe zone.

* Geometric construction, not illustrative.

* One colour at a time, drawn from the palette.

## **Default colours**

* Loyal Navy on light surfaces.

* Royal Red for active, emphasised, or destructive states.

* Clermont Cream on dark surfaces.

* Dusty Blue for muted or inactive states.

## **Library**

Lucide ([lucide.dev](https://lucide.dev)) is the open-source baseline. Custom icons match Lucide in geometry, weight, and corner radius so a mixed set still reads as one family.

## **Avoid**

* Filled or solid icons, unless briefed for status indicators.

* Multicolour or gradient icons.

* Skeuomorphic or photo-realistic icons.

* Inconsistent stroke weights within a single layout.

**SECTION 09**

**Imagery and photography**

Imagery brings AERIS down to earth. It shows the people and places the platform serves, doing the work AERIS supports.

## **Direction**

Real Australian businesses across trades, retail, hospitality, agriculture, and services. Working environments: workshops, yards, paddocks, cafes, warehouses, shopfronts, processing sheds, office floors.

* Natural light. Neutral grading. Honest expressions.

* Wide and medium shots that show context.

* Tighter detail shots that show the work: hands, machinery, products, screens.

* Real people who actually do the job.

## **Treatment**

* Slight warm bias for outback and rural contexts.

* Generous crops. The frame should breathe.

* Brand colour overlays for editorial moments: Loyal Navy at 80%, Clermont Cream at 60%. The overlay supports the photo, it doesn’t replace it.

## **UI screenshots**

When AERIS UI appears in marketing, frame it simply. A subtle drop shadow is fine. Avoid 3D laptop mockups, perspective tilts, and floating screens disconnected from any device. Where possible, show the product running in a real environment.

## **Avoid**

* Stock corporate imagery. Handshakes in suits. Glass towers. Abstract tech.

* Heavily filtered, oversaturated, or HDR photos.

* Empty environments with no people or product.

* Models who look like models.

* Aerial drone shots with no human scale.

**SECTION 10**

**Buttons and calls to action**

AERIS buttons signal the next action. They’re not decoration. One primary button per view. Never two primary buttons fighting for attention.

## **Primary**

* Filled in Red Dirt Red. White text.

* Poppins SemiBold.

* Rounded pill or 8 px corner radius.

* Horizontal padding 1.5 to 2 times the vertical.

## **Secondary**

* Outlined in Loyal Navy. Navy text. No fill.

* Same dimensions as primary.

## **Tertiary**

* Loyal Navy text. Optional underline on hover.

## **Destructive**

* Filled in Royal Red. Used only for irreversible actions.

## **Labels**

Verb-led. The label says what the button does.

* **Book a demo**, not *Click here*.

* **Download report**, not *Submit*.

* **See pricing**, not *Learn more*.

* **Start free trial**, not *Get started*.

**SECTION 12**

**Gut check**

Before any AERIS visual goes out, run it through these five questions. If the answer to any of them is no, rework it.

| 1\.  Does it use only the brand palette? 2\.  Is the typography limited to Poppins? 3\.  Does it have enough whitespace for the eye to rest? 4\.  Would it sit comfortably alongside other AERIS work? 5\.  Could you defend every design choice as serving a purpose? |
